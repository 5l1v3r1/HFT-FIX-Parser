Welcome on FIX Nano Project

This tool written in Java aims at providing the best latency so far for high frequency trading message processing activities.

As a first run, you may launch it this way :

$ cd philipperemy/fix
$ git fetch origin
$ git checkout master
$ java -server -jar fix-nano.jar

For JVM extreme optimizations, please refer to this link : http://javolution.org/

Message Processing

A set of functions has been crafted to deal with message processing.

INPUT : FIX Message OUTPUT : Map fields

The latency is on average 350ns from receiving the FIX message and filling the fields map.
